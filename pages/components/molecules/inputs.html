Estados

Default

Estado inicial del campo.

Active

Campo activo con interacción.

Typing

El usuario está ingresando datos.

Filled

El campo contiene datos válidos.

Disabled

Campo no editable.

Message

Error

Indica un error en la validación.

Variantes

Sin icono

Variante estándar para entradas de datos simples.

Icono a la izquierda

Ideal para identificar el tipo de dato como un email.

Icono a la derecha

Utilizado frecuentemente para selectores o menús.

Utiliza letras, números y puntos.

Con mensaje

Incluye texto de ayuda para guiar al usuario en el ingreso de datos.

Sin mensaje

Variante compacta cuando no se requiere información adicional.

Accesibilidad

Etiquetado

Cada campo debe tener una etiqueta descriptiva asociada mediante el atributo `for` o `aria-labelledby`.

Contraste

Garantizar que el texto y los bordes tengan suficiente contraste con el fondo (mínimo 4.5:1).

Navegación

El orden del foco debe ser lógico y predecible. Todos los estados de foco deben ser visibles.

Lectores de pantalla

Utilizar `aria-invalid` y `aria-describedby` para comunicar errores y mensajes de ayuda.

Buenas Prácticas

Qué hacer

  • Usar placeholders solo como guía, no como reemplazo de etiquetas.
  • Agrupar campos relacionados de manera lógica.
  • Utilizar máscaras de entrada para datos con formato específico.

Qué evitar

  • Evitar el uso de demasiados campos requeridos innecesarios.
  • No ocultar información crítica dentro de tooltips.
  • Evitar el autocompletado en campos de seguridad si no es necesario.
