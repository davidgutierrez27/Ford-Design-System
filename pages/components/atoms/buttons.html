Variantes

Utilizamos distintos niveles de énfasis, tamaños y colores para comunicar la jerarquía de las acciones.

Jerarquía

Primario (Primary)

Para la acción más importante de la pantalla. Debe destacar sobre los demás elementos.

Acción principal

Secundario (Secondary)

Para acciones importantes que no son la principal (ej. "Cancelar", "Atrás", filtros).

Acciones alternativas

Terciario (Tertiary)

Para acciones con la menor prioridad o enlaces dentro de contextos no intrusivos.

Menor importancia

Implementación

Utiliza las clases utilitarias para implementar botones rápidamente.

Clases Base

Combina clase base + tamaño + tema + variante.

<!-- Estructura Básica -->
<button class="btn btn--medium theme--skyview--primary">
   Label
   <svg class="btn--icon" ...></svg>
</button>

<!-- Modificadores de Tamaño -->
.btn--small
.btn--medium
.btn--large

<!-- Temas -->
.theme--skyview--primary | secondary | tertiary
.theme--grabber--primary | secondary | tertiary
.theme--neutral--primary | secondary | tertiary

<!-- Estado -->
.btn--disabled

Accesibilidad y Buenas Prácticas

Garantizar que los botones sean accesibles es fundamental para una buena experiencia de usuario.

Mejores Prácticas

  • Labels descriptivos: Usa verbos activos (ej. "Enviar", "Guardar") en lugar de genéricos ("OK", "Clic aquí").
  • Área de clic: Asegura un tamaño mínimo de 44x44px para interacciones táctiles (o añade padding transparente).
  • Estado Focus: Nunca elimines el outline de focus sin proporcionar un estilo alternativo visible para navegación por teclado.
  • Disabled: Evita usar botones deshabilitados si es posible; es mejor mostrar un error al hacer clic explicando por qué no se puede proceder.

Contraste

Los botones deben tener suficiente contraste contra el fondo y entre el texto y el color del botón.
