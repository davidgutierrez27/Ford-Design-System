Principios

Funcionamiento

El color se utiliza para comunicar significado, jerarquía y estado, no como ornamento visual. Cada elección debe tener un propósito claro.

Identidad

La identidad de marca se concentra en pocos colores clave, usados de forma consistente para fortalecer el reconocimiento y evitar ruido visual.

Escalabilidad

Las escalas cromáticas permiten adaptarse a distintos contextos sin romper la coherencia visual, funcionando en experiencias simples y complejas.

Accesibilidad

Garantizamos que el color nunca sea el único medio de comunicación, asegurando contraste y claridad para todos los usuarios.

Mapeo de Tokens y Uso Recomendado

Estos colores deben consumirse a través de su token semántico para permitir cambios futuros de paleta a nivel global.

  • --color-status-error Validación negativa, mensajes de falla crítica, errores de servidor o entrada.
  • --color-status-success Acción completada exitosamente, validación positiva de formularios.
  • --color-status-warning Alertas que requieren atención pero no bloquean la acción (ej: baja batería, límite de uso).
  • --color-status-info Información neutral para el usuario (ej: un tip, una nueva característica).

Implementación

Utiliza siempre tokens semánticos (CSS variables) para mantener consistencia y soportar temas futuros. El uso directo de valores hexadecimales está desaconsejado.

Buenas Prácticas

Do: Usa Tokens Semánticos

Referencia siempre variables CSS para permitir cambios globales.

.btn-primary {
  background: var(--color-primary-500);
  color: var(--color-text-white);
}
Don't: Hardcode Hex Values

Evita usar valores crudos que rompen la escalabilidad.

.btn-primary {
  background: #066fef; /* ❌ No escala */
  color: #ffffff;
}

Convención de Nombres

Nuestra escala numérica indica el propósito y peso visual del color.

  • 0-100: Fondos, superficies claras y bordes sutiles.
  • 200-400: Bordes de inputs, iconos inactivos y textos deshabilitados.
  • 500: Color BASE. Usado para acciones principales y estados de reposo.
  • 600-700: Estados Hover y Active.
  • 800-900: Textos de alto contraste y bordes de foco.

Accesibilidad y Contraste

El color no es solo estética; es funcionalidad. Aseguramos que nuestros colores cumplan con los estándares WCAG 2.1 AA para contraste y distinción.

Contraste de Texto

Todo texto debe tener suficiente contraste contra su fondo para ser legible por personas con baja visión.

Checklist de Diseño

Verifica estos puntos antes de finalizar cualquier diseño o componente.

  • No solo color: ¿Usas iconos o texto además del color para errores/éxito?
  • Links visibles: ¿Los enlaces se distinguen del texto normal (subrayado/peso)?
  • Estados de foco: ¿Es visible el anillo de foco (Focus Ring) al navegar con teclado?
  • Modo Oscuro: ¿Has validado que los tokens semánticos funcionen en dark mode?
