Principios de Spacing

La Regla del 4

Nuestra sistema se basa en un grid de 4px. Todos los valores son múltiplos de 4, asegurando alineación y proporción matemática.

Jerarquía Espacial

El espacio vacío no es solo "aire", es una herramienta para agrupar o separar contenido y establecer niveles de importancia.

Ritmo Vertical

Al usar una escala predefinida, creamos un ritmo visual constante que facilita el escaneo de la página por parte del usuario.

Respiración (White Space)

El uso generoso del espacio reduce la carga cognitiva y permite que el contenido principal destaque sin distracciones.

Uso en Layout

El espaciado de layout controla los grandes bloques de la interfaz y la separación entre secciones principales.

Secciones Principales

Usamos valores grandes (64px - 96px) para dar "aire" entre secciones de una misma página.

  • Separación entre Hero y Contenido
  • Gaps entre módulos de Dashboard
  • Márgenes laterales de contenedor principal

Uso en Componentes

El espaciado de componentes (padding y gap) define la densidad de los elementos interactivos.

Padding Interno

Define cómo respira el contenido dentro de un componente como una card o un botón.

  • Cards: suelen usar 24px o 32px de padding.
  • Botones: usan espaciado asimétrico (ej. 8px vertical, 16px horizontal).
  • Listas: usan 12px o 16px entre elementos.

Implementación

Utiliza siempre los tokens de espaciado. Nunca uses valores de píxeles "hardcoded" en tus archivos de estilo.

Tokens Destacados

Los tokens más utilizados en el día a día.

Token Valor Uso común
var(--space-16px) 16px Padding estándar / Gaps pequeños
var(--space-24px) 24px Padding de cards / Gaps medianos
var(--space-80px) 80px Separación entre secciones (Layout)

Ejemplo SCSS

Aplicación de márgenes y paddings con tokens.

.section-container {
    padding: var(--space-48px) var(--space-24px);
    margin-bottom: var(--space-80px);
}

.card-header {
    display: flex;
    gap: var(--space-12px);
    padding-bottom: var(--space-16px);
}
