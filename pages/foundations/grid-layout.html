Principios de Layout

Proporcionalidad

El uso de una grilla de 12 columnas permite que los elementos mantengan una jerarquía visual clara y proporcional en cualquier dispositivo.

Adaptabilidad

Nuestra estructura responde automáticamente a los cambios de resolución, redistribuyendo el contenido para optimizar el espacio.

Consistencia

Márgenes y gutters estandarizados garantizan que el alineamiento sea idéntico en todas las secciones del producto.

Ritmo Visual

El espaciado sistemático crea un flujo de lectura predecible y confortable para el usuario.

Resoluciones y Breakpoints

Definimos cuatro puntos de interrupción principales para cubrir todo el espectro de dispositivos, desde móviles hasta monitores ultra-wide.

Dispositivo Ancho (px) Columnas Márgenes Gutter
Desktop Wide 1440+ 12 160px 24px
Desktop 1024 12 24px 24px
Tablet 768 8 24px 24px
Mobile <768< /td> 4 16px 16px

Visualización técnica de la distribución de columnas según el punto de interrupción activo.

Zonas Seguras

Los márgenes laterales aseguran que el contenido nunca toque los bordes físicos de la pantalla, manteniendo el foco en el centro.

Márgenes (Margins)

Espacio fijo entre el contenedor principal y el borde del viewport. Varía según el dispositivo para optimizar el ancho de lectura.

Medianiles (Gutters)

Espacio entre columnas. Evitan que el contenido de bloques adyacentes se mezcle visualmente, facilitando la separación de conceptos.

Media Queries (Mixins)

Utilizamos mixins de SASS para gestionar los puntos de interrupción de forma centralizada y semántica.

@use "media" as media;

// Desktop Wide (1440px)
@include media.xl { ... }

// Desktop (1024px)
@include media.lg { ... }

// Tablet (768px)
@include media.md { ... }

// Mobile (480px)
@include media.sm { ... }

Container Queries

Para componentes modulares, utilizamos Container Queries. Esto permite que un elemento responda al tamaño de su padre en lugar del viewport.

// Definir el contenedor
.parent { container-type: inline-size; }

// Mixin para el hijo
@include media.cq-md {
  display: flex;
  flex-direction: column;
}

Buenas Prácticas

✓ Qué hacer
  • Alinea siempre los bloques principales a las 12 columnas.
  • Usa el mixin de media query correspondiente al breakpoint.
  • Prioriza Container Queries para componentes reutilizables.
  • Mantén gutters consistentes en toda la aplicación.
✗ Qué evitar
  • No uses valores de ancho fijos (px) para contenedores.
  • Evita saltar breakpoints (ej. pasar de XL a SM directamente).
  • No coloques contenido crítico fuera de la zona segura.
  • Evita anidar demasiadas grillas, simplifica la estructura.
