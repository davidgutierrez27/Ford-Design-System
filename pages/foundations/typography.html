Principios Tipográficos

Consistencia

Mantener patrones reconocibles a través de todas las interfaces asegura que los usuarios puedan navegar y comprender el contenido sin esfuerzo adicional.

Jerarquía

Permitir distinguir títulos, subtítulos y cuerpo de forma inmediata mediante tamaño, peso y espaciado diferenciado, guiando la lectura.

Escaneo rápido

Mejorar cómo los usuarios consumen texto en pantallas mediante interlineado adecuado y longitudes de línea óptimas para reducir la carga cognitiva.

Flexibilidad

Adaptarse a distintos tamaños de dispositivos manteniendo legibilidad y jerarquía visual en todas las resoluciones y contextos de uso.

Anatomía de la tipografía

Cada estilo tipográfico tiene atributos definidos: tamaño, peso, interlineado y familia. Estos parámetros garantizan lecturabilidad y consistencia en la interfaz.

Componentes que definimos

  • Tamaño (font-size): Expresa el nivel de importancia visual. Usamos una escala de 12px a 54px para cubrir todos los casos de uso.
  • Peso (font-weight): Diferencia énfasis y roles. Disponemos de Regular (400), Medium (500) y Bold (700).
  • Altura de línea (line-height): Mejora la lectura en bloques largos. Usamos ratios de 1.1 (tight), 1.3 (normal) y 1.5 (relaxed).
  • Familia tipográfica: Arial sans-serif establece la voz del producto con una fuente legible y profesional.

Roles Tipográficos

Asignamos roles claros a cada estilo para evitar uso incorrecto y mantener coherencia. Cada rol tiene características específicas y casos de uso definidos.

Títulos (Headings)

Se utilizan para secciones, encabezados principales y contenido destacado. Establecen la jerarquía principal del contenido.

Implementación

Todos los estilos del sistema se implementan mediante mixins SCSS para asegurar consistencia, reutilización y mantenibilidad. Cada mixin encapsula tamaño, peso, line-height y familia tipográfica.

Ejemplos de Uso

Patrón recomendado para implementar tipografía en archivos SCSS.

// 1. Título de sección
.section-title {
    @include heading-title-32-regular; /* Aplica estilos de H3 */
    color: var(--color-text-900);
}

// 2. Texto de contenido
.article-text {
    @include body-16-regular; /* Aplica Body 16 con LH 1.5 */
    color: var(--color-text-500);
    max-width: 600px;
}

// 3. Botón personalizado
.btn-primary {
    @include button-14; /* Aplica weight bold y uppercase si configurado */
    color: var(--color-text-white);
}

Accesibilidad y buenas prácticas

La tipografía debe ser legible para todos los usuarios. Seguimos las pautas WCAG 2.1 para garantizar accesibilidad en contraste y legibilidad.

Contraste de Color

Asegura un ratio mínimo para garantizar la lectura.

✓ Cumple AAA (Ratio 15.8:1)

Texto negro (#262626) sobre blanco

✓ Cumple AAA (Ratio 15.8:1)

Texto blanco (#fefefe) sobre negro

✗ No cumple AA (Ratio 2.4:1)

Gris claro (#b5b5b5) sobre blanco

Mejores Prácticas

Reglas de oro para una tipografía accesible.

  • Contraste mínimo AA: 4.5:1 para texto normal, 3:1 para texto grande.
  • No usar solo color: Combinar color con peso o iconos para transmitir información.
  • Interlineado óptimo: Al menos 1.5 veces el tamaño de fuente para párrafos.
  • Longitud de línea: Máximo 60-80 caracteres por línea para evitar fatiga.
  • Jerarquía clara: No saltar niveles de encabezado (H1 -> H2 -> H3).
  • Fuentes relativas: Usar 'rem' o 'em' para respetar preferencias del usuario.
